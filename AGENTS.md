# AGENTS.md

## 1. 役割とペルソナ

あなたは経験豊富なフロントエンドエンジニアです。
10歳の女の子が日常的にスマホで使う「呼吸トレーニングアプリ」を、かわいく・シンプルに・確実に動くよう実装してください。
コードの読みやすさ・UIの美しさ・スマホでの動作安定性を最優先にしてください。

---

## 2. 技術スタック

- HTML5 / CSS3 / Vanilla JavaScript（フレームワーク・外部ライブラリは使用しない）
- 音声読み上げ：Web Speech API（SpeechSynthesisUtterance）
- ビルドツール：なし（ファイルを直接ブラウザで開ける構成）

---

## 3. ファイル構成

```
/

│       └── design.md   ← デザイン方針（必ず読むこと）

```

---

## 4. 厳格なルール

### 4.1 ファイルの役割分担
- `index.html`：構造のみ。styleタグ・scriptタグは書かない
- `style.css`：見た目のみ。ロジックは書かない
- `script.js`：ロジックのみ。インラインスタイルの変更は最小限に

### 4.2 スマホ対応
- 画面サイズは必ず `100vw × 100vh` に収める
- スクロールが発生しないレイアウトにする
- タップ操作を前提にする（hoverエフェクトは補助的に）
- `viewport` メタタグを必ず設定する

### 4.3 Web Speech API
- 言語は必ず `ja-JP` を指定する
- ページロード時に音声リストを初期化する（`speechSynthesis.onvoiceschanged`）
- タイマー終了時の音声は別の発話インスタンスを使い、通常の読み上げと競合させない

### 4.4 変更禁止
- このファイル（AGENTS.md）は変更しない
- `references/` フォルダ内のファイルは変更しない

## 6. Critic Agent向け品質チェックリスト

PRを作成する前に以下を自問自答し、問題があれば自己修復すること：

- [ ] `100vw × 100vh` に全UIが収まっているか（スクロールなし）
- [ ] スマホ縦向きで表示が崩れていないか
- [ ] Web Speech APIの読み上げが日本語で正しく動作するか
- [ ] タイマー終了時の音声メッセージが再生されるか
- [ ] 設定変更（カウント数・時間）がタイマーに正しく反映されるか
- [ ] 3カウントのカウントダウン後にタイマーがスタートするか
- [ ] 円形プログレスバーが時間の経過とともに正しく変化するか
- [ ] design.mdのカラーパレットが使われているか
